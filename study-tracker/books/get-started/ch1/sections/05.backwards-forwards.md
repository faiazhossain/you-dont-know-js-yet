### Best Practices for Compatibility

- Write code using the latest stable JS features
- Let tools handle backwards compatibility
- Don't manually adjust for syntax/API gaps - use transpilation and polyfills
- Focus on clean, readable code that communicates intent effectively
- ***

### Jumping the Gaps (Transpilation)

- Since JS is not forwards-compatible, new syntax will break in older engines
- **Transpiling**: Converting source code from one version to another (still as text)
- Babel is the most common transpiler
- Example: Converting `let` (ES6) to `var` (ES5) for older browsers
- Before:
  ```javascript
  if (something) {
      let x = 3;
      console.log(x);
  } else {
      let x = 4;
      console.log(x);
  }
  ```
- After transpilation:
  ```javascript
  var x$0, x$1;
  if (something) {
      x$0 = 3;
      console.log(x$0);
  } else {
      x$1 = 4;
      console.log(x$1);
  }
  ```
- Developers should write modern, clean syntax and let tools handle compatibility
- The "target" for transpilation shifts upward as we drop support for older browsers
- ***

### Filling the Gaps (Polyfills)

- **Polyfill** (aka "shim"): Providing a definition for missing API methods
- Used when the issue is a missing API, not new syntax
- Example: ES2019's `Promise.prototype.finally(..)` in pre-ES2019 environments
- Polyfill pattern:
  ```javascript
  if (!Promise.prototype.finally) {
      Promise.prototype.finally = function f(fn) {
          return this.then(
              function t(v) {
                  return Promise.resolve(fn()).then(function t() {
                      return v;
                  });
              },
              function c(e) {
                  return Promise.resolve(fn()).then(function t() {
                      throw e;
                  });
              }
          );
      };
  }
  ```
- The `if` statement prevents overwriting in environments where the method exists
- Transpilers like Babel typically detect and auto-include needed polyfills
- Always use robust, official polyfills (e.g., from ES-Shim) rather than writing your own
- ***

## Code Examples

```javascript
// Examples showing compatibility considerations

// Backwards compatibility: Code from 1995 still works today
// Old code doesn't break when browsers update

// Forward compatibility issue (NOT supported in JS):
// New syntax in old engine = Syntax Error
const x = 5; // Fails in pre-ES6 browsers

// Solution: Transpile with Babel
// Source: const x = 5;
// Output: var x = 5;

// Missing API issue:
Promise.prototype.finally(..); // Doesn't exist in pre-ES2019

// Solution: Polyfill
if (!Promise.prototype.finally) {
    // Provide the implementation
}
```
